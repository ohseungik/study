# Design System

이 프로젝트의 디자인 시스템 문서에 오신 것을 환영합니다.

## 개요

이 디자인 시스템은 **일관성**, **재사용성**, **접근성**을 목표로 구축되었습니다.

## 주요 원칙

### 1. 일관성 (Consistency)
모든 컴포넌트는 동일한 디자인 토큰을 사용하여 일관된 사용자 경험을 제공합니다.

### 2. 재사용성 (Reusability)
각 컴포넌트는 독립적으로 작동하며, 다양한 컨텍스트에서 재사용할 수 있습니다.

### 3. 접근성 (Accessibility)
모든 컴포넌트는 WCAG 2.1 AA 기준을 충족하도록 설계되었습니다.

## 시작하기

### 컴포넌트 사용법

```tsx
import { Button } from '@/components/ui/Button';

function App() {
  return (
    <Button variant="primary" size="medium">
      클릭하세요
    </Button>
  );
}
```

### Storybook Controls

각 컴포넌트 스토리에서는 **Controls** 탭을 통해 실시간으로 props를 변경하여 
컴포넌트의 동작을 테스트할 수 있습니다.

## 컴포넌트 목록

현재 제공되는 컴포넌트:

- **Button**: 다양한 스타일과 크기의 버튼
- **Input**: 폼 입력 필드
- **Card**: 컨텐츠 그룹화를 위한 카드

## 기술 스택

- **React 19**: 최신 React 기능 활용
- **TypeScript**: 타입 안정성
- **Tailwind CSS**: 유틸리티 우선 스타일링
- **Storybook 8**: 컴포넌트 문서화 및 테스트

## 문서 구조

- **Introduction**: 디자인 시스템 소개 (현재 페이지)
- **Guidelines**: 컴포넌트 사용 가이드라인
- **Tokens**: 디자인 토큰 정의
- **UI Components**: 각 컴포넌트의 상세 문서
